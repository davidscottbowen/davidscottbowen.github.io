Swift Lesson 1

Constants and Variables

Posted by David Scott Bowen on April 20, 2017

Any Swift variable is either a constant or not. Not to be confused with the type of the variable, such as Float or Int, constants and variables are just ways to describe terms that hold a value which can change (are mutable), and constants that can not be changed (becuase they are immutable).

To define a constant, use let .

Example:

let name = "David"

If you were to try and change the value of name, you would be unable to do so, and the Swift compiler would produce an error.

let name = "David"

name = "Scott"

error: cannot assign to value: 'name' is a 'let' constant

name = “Scott"

~~~~ ^

The swift complier suggests how to correct this error.

note: change 'let' to 'var' to make it mutable

let name = “David"

^~~

var

By using var, we define a variable that can change:

var name = “David"

name = “Scott"

Immutability is preferred over mutability, thats why you should always default to using let , unless you know you need a var . This leads to safer code. If you define a constant, and later attempt to modify it, you will get an error and at that time you can determine if you should switch to using var.
